# TYPE 목록
# FEAT      : 새로운 기능 추가
# FIX       : 에러 및 버그, 기능 수정
# DOCS      : 문서 작성 및 수정
# STYLE     : 코드 포맷팅 (비즈니스 로직 변화 없음)
# REFACTOR  : 코드 리팩토링 (기능 변화 없음)
# TEST      : 테스트 관련 작업
# CHORE     : 빌드 설정 등 기타 변경 (기능 변화 없음)
# RENAME    : 파일 이름 또는 변수명 변경
# REMOVE    : 파일 및 코드 삭제
# MERGE     : 브랜치 머지 작업
# COMMENT   : 주석 추가 및 수정
# INIT      : 프로젝트 초기 생성

# 제목 규칙
# - 한 줄로 작성하며, 최대 50글자가 넘지 않는다.
# - 마침표 및 특수기호는 사용하지 않는다.
# - 개조식 구문으로 작성한다. (완전한 서술형 문장이 아닌, 간결하고 요점적인 서술)
# - 필요 시 괄호 안에 관련 이슈번호를 적는다.

# 본문 규칙
# - 본문은 선택적으로 작성하며, 제목과 한 줄 띄어 구분한다.
# - 어떻게 보다는 "무엇을", "왜" 했는지를 작성한다.
# - 여러 줄일 경우 문단 구분한다.

# 커밋 예시
# [FEAT] 게시글 좋아요 기능 추가 (#12)
#
# 사용자가 게시글에 좋아요를 눌러서 상태를 저장할 수 있도록 기능을 구현했습니다.
# 로컬 상태와 서버 동기화를 포함합니다.
#
